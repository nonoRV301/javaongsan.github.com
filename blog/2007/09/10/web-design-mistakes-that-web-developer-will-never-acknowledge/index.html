Web Design Mistakes That Web Developer Will Never Acknowledge

Sep 10th, 2007

Web designer, developer and coder pride themselves as Gog of the Internet, creator of things digital. They design website that uses the latest technology, fancy scripts, colorful moving objects, out-of-the-world flash but they can made a single cent on the web and THATS THE TRUE! How do I know? I have been a coder and have work the Internet for over 10 years and till recenlty I wasn’t making any money for myself or my clients. I list out the most common mistake a professional web designer are mostly to made.

  1. Using Flash Template as a website.

If you still living in 1990s, this is fine but Web 2.0 are already here, so wake up and smell the digital coffee. Flash templates are not interactive and they will never be. They are just artistic piece of art that moves around on the web. When you need a web designer, always ask for a PHP (a open source software which means this tool is free) programmer who knows web design and not a web designer who can build a website.

  1. Using Flash on your website.

I have nothing against flash but even a tiny little flash program running on your web page can cause the amount of loading time your entire web page add together. Let me put it this way, “Flash is not cash”.

  1. Using too many colours.

A basic website should only use a maximum of 4 colors. Background, Text, Link are most common identified color areas. There’s no need to be colorful, just think of a book, amgazine or newspaper you read recently, what the color of the text and whats the color of the background.

Using Dark Background.

The dark background are used a lot by hacker sites back in the 80s. White text on Black background may look really cool, it does not work for a normal website that is looking to attract consistent readers.

  1. Using too many scripts.

Scripts like javascript, ajax, vbscript are common on web page to perform task that uses recourse from client machine and to take the load of the server. However too many scripts too will slow the whole web page down. You would want to remove fancy scripts that does not i any way benefit the user.

The cute redirection.

One of the worst script around is the redirection javascript. If you are using one, I beg you, please remove it. This script says “Please wait while we redirects you in a few seconds”. What happen if its does not work? This script will sent most of your visitor away before your script finish loading your site.

  1. Inconsistent navigation style.

What can be worse than not able to find the link that you need? So many website uses fancy menu that pop in and out, and some will not stand still long enough for my mouse to click on them.

  1. Fonts are too big or too small.

The font fraorite by designers and coder are a little smaller than what mormal web surfer prefer. We used to smaller fonts to see more codes and we use too it. We used fonts like courier that no one likes.

  1. Not providing help or FAQ for users.

Most designer will design the website but they will not take time to put in a help or faq on how to used the site, especially f its a community or forum. Even of a experience user it takes time to know what the website is trying to do and how to perform the action we want.
