How to Control Mdi Form Menu Dynamically for Different Login User

Jan 19th, 2014 | Comments

Controlling winform for multi-user application can be easy achieve by limiting menu access based on login user. Most of the time, a role is required because many user have the same access rights. We can either add this manually or have everything store in an database.

```

foreach (ToolStripItem item in menuStrip1.Items)
{
    item.Enabled = RoleAccess.IsEnable(RoleId);
    item.Visible = RoleAccess.IsEnable(RoleId);
}

```

Building Your Own Software Product

Jan 18th, 2014 | Comments

This is not a guide, I do not have a profitable product yet, This is a collection for my own references.

If you been freelancing for a while, chances are you get the itch sometimes to create your own software product. For a freelance programmer, the best way to create recurring income is to create your own software product.

How do we go about creating an software product. First, find your niche. A niche is a group of people with the same behavior, in software sense, they use the same type of software to perform a common task.

When you have selected a niche, its time to do some research. Research your customers, look at what products they own, how much are they paying for. Go to forums, user groups, on facebook, twitter and linkedin. Find out what are their problems, pains, desires.

Generating Ideas for software.

Automated a time consuming task, build a companion to a task they need to automate e.g. a application that replace then need for a user to manually checking website related to their trade that do not provide filtering or email alerts.

Another great way to build software is to start out by solving your own problems. find other people who had the same problem. Look at what they are doing to solve it.

Look at what you are currently doing and what are your customer paying you to solve. Can some one else use it?

If someone have ask you about a certain kind of solution more than once, start taking notes.

Build your prototype, use agile methodology, start small learn and iterate. Then you’ll be able to build a product your customer truly love.

For the next 30 days, find at least one problem per day. Record it.

Should You Work for Equity Instead of Cash?

Jan 17th, 2014 | Comments

I been asking a few times to take equity instead of cash to help with the development of mostly startups or those who wants to do start up. I would definitely take equity if a right project I like comes along.

My rules are : 1. If project have good potential and generates income right form the start, I take equity. 2. If project have good potential and may generate income in distant future, I take small equity plus small upfront cash. 3. If project have no potential, I charge full development cost. 4. Reject anything else thats a waste of time.
